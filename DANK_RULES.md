## 🚀 Build Previews & Rendering (Rule 1)
- **Automatic Loading**: All builds must display the model instantly in the preview window without manual configuration.
- **Consistency**: Preview behavior must be uniform across all sections (Bunkers, Traders, Arenas, etc.).
- **Fixes**: If a preview fails, it must be debugged and fixed immediately to ensure a seamless experience.

## 🌎 Real-World Accuracy (Rule 2)
Before finalising ANY build:
1.  **Google Research**: Search for the exact name in quotes (e.g., `"AT-AT Walker"`).
2.  **Analysis**: Review multiple real-world images from Google Images.
3.  **Accuracy Checklist**: Ensure the build matches the reference in:
    - **Shape & Proportions**
    - **Color Palette**
    - **Layout & Positioning**
    - **Structural Details**

## 📏 Part Scaling & Placement (Rule 3)
- **Scale Awareness**: You must understand the exact physical size of every part.
- **Solid Structures**: Overlap pieces correctly to create gap-free, solid structures.
- **Realistic Spacing**: No gaps between pieces that should be connected. No unrealistic distancing (e.g., a 1m part cannot have a 5m gap).
- **Alignment**: **ZERO** floating parts, misalignment, or structural gaps.

## 🏗️ Structural Integrity (Rule 4)
- **Unified Models**: Every build must appear as a complete, unified structure.
- **Proportions**: Maintain consistent proportions across the entire model.
- **Logical Placement**: Components must be placed where they make sense in a real structure. No empty spaces where a real structure would be solid.

## ✅ Quality Control Checklist (Rule 5)
Before each build is marked "Complete", it must pass:
- [ ] Preview loads automatically
- [ ] Matches real world (Google Image Research)
- [ ] Zero gaps between parts
- [ ] Correct part sizes used
- [ ] Realistic spacing
- [ ] Accurate placement
- [ ] Structure is complete and unified

## 🧹 Maintenance (Rule 6)
- **Updates**: Always correct inaccurate information. 
- **Purge**: Remove outdated or irrelevant details.
- **Consistency**: Apply these standards to every single piece of data and code.

## 🛡️ Self-Healing Immutability (Rule 7)
- **Zero-Geometry Protection**: No build should ever cause a crash or white screen due to missing points. Fallbacks must be implemented at the pipeline and renderer levels.
- **Additive Development**: Never delete or revert working systems. Only patch, wrap, or extend them to ensure backward compatibility.
- **Automated Audits**: A full architectural compliance audit (`auditAllBuilds`) must run on every application bootstrap to identify and report structural anomalies.
